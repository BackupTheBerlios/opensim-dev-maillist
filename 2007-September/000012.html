There is an interesting issue with the current OpenSim base, where you
are often either slightly hovering above the ground, or crouching.  The
root cause of this appears to be that OpenSim is only storing heigh maps
as 1m resolution grids, where the client is seeing something much more
detailed than that (not sure if it is interpolation client side, or
there is more data there).

Adam, as you have been the master of the terrain space, is it possible
to make the height map arrays at 0.1m resolution?  Or should we consider
some interpolation for this data?
