Almost all of the individual server can potentially load an array of 
plugins. In the case of the grid server, it will be possible to attach two 
different grid server plugins, e.g. one for deepgrid and one for osgrid.

Lets say we have that scenario, and the user logs into osgrid, then 
teleports to a place which isn't on osgrid, but the plugin chaining 
redirects to deepgrid, now the user has moved deepgrid, probably without 
knowing it. so lets say deepgrid has two sims

LiveConcert and Sandbox

OSgrid also has two sims
ArtDisplay and Sandbox

in my scenario the user is in "ArtDisplay" and teleports to "LiveConcert", 
since we - in this scenario - have two plugins in the grid server, the tp 
will work, and the user ends up in "LiveConcert"

so what happens to the users contact list, im's, what happens to name2id 
lookup, in case of two av's with identical names?, what will a consistent 
experience be?

lets say the user tp's to Sandbox, what grid will he end up in?
