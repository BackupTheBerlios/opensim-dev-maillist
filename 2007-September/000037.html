I think it would be good to discuss a little bit on what people think
are important for 0.5, what is the criteria for 0.5, and what are some
things we should dig into early?

On the "big list of goals"
* Physics
* Scripting
* Grid Mode more stable
* Multiple database writers (something needed for create account php
  page for osgrid and others).

On the "list of pernicious issues in the code base"
* Windows server crash when logging off of different region than logging
  on
* Can't unlink prims
* Can't do certain inventory updates
* Avatar facing the wrong direction when altering appearance

What else are on people's todo lists?  Please respond with discrete
points, and we can create an agregate list.

Much like 0.4, I think we want to put a stake in the ground time wise
(say early November), and as we get closer to the date figure out what
is looking good, and that we can wrap up for the release, and what
pushes off past it.  If something big (like physics) comes together
early, have an early release wouldn't be bad either.

Hopefully such a list will also help us draw in more folks from #opensim
to the development process, as there are some decrete things we want
fixed, and some discrete places people can start.
