Hello,

I'm looking into the logging system that OpenSim uses.  Right now it
seems that all the servers use the Framework.Console.LogBase class to
handle their logging needs.  It would be nice to have some configuration
options, however, for things like message format and destination (file,
database, etc.).  Apache's log4net project seems like an interesting
option.  Are there any reasons not to use such a library?  Also, log4net
is licensed under the Apache license ver. 2.0, but would that be a
problem?
