I'm not sure what that means exactly.  I think if we don't have a single
command that is run that builds the whole tree (including externals) it
doesn't win us enough.

is the wrong approach for anything complicated, and not something I'm
really interested in spending much more time on.

I really think a single unified build system will help a lot with the
project moving forward, especially as we get folks out on new platforms
and architectures.
