You misunderstood me. I know for a fact that the onus of 
administering a separate user base would turn a number of grid 
operators I know off using it.

There definitely needs to be a setup that takes all logins and 
levels from the grid database and requires zero admin if people join 
or leave, other than adding them to the grid (user server) database.

I don't know why you would reject using the data that is already 
there, but the option should be there for those who want it.

Melanie
