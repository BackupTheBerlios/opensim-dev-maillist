Hello all
i'm currently working on some gridlevel commands.

I already wrote some grid commands :

grid-offline : set an offline mode for the region registration: A new region
won't be able to register, but the already running ones are not kicked out.
This is usefull to test some release without "outworld" perturbation
grid-online :: set the grid back to a regular operation : allowing new grids
to register
grid-status : display the actual status.

The mantis and the associated patch will be delivered this afternoon.

What i would like to implement now :

Grid-kicking : This command will kick out the named region or all the region
Grid-filtering : This command will only allows a region with a certain
release to register. The command is similar to the minimal userlevel already
implemented, but extended to a region. The filter should be the simulator's
release number. That will prevent not updated sim to jam the system.

For that, i will need to add a field "release" into the region's db.

any thought?

Sacha
