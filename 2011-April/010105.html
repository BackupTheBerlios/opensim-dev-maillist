There has been a few changes in recent 0.7.1 dev master releases for 
the way in which a hypergrid linked region is labelled and accessible 
when seen on a map or in map search.

I note today when linking the Newton region on ScienceSim with this 
command in Robust.exe console...

link-region xloc yloc http://grid.sciencesim.org/grid./hypergrid.php Newton


It shows as name "Newton".. on the map, and in show hyperlinks 
command in the Robust.exe console with nothing to indicate it is from 
grid.sciencesim.org

Earlier link-region commands in the same setup have long names that 
fully include the loginuri.

Going back to 0.6.9 we used to be able to give a string that would be 
used locally.

Is there a way to customize the name given in the map so the string 
can indicate the grid name, or a string chosen can be allocated.  Or 
at least we should make sure that the show hyperlinks command 
continues to indicate the loginuri/grid its from to make it more manageable.

What is the aim and plan for this?  Perhaps Diva can comment?
