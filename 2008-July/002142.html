One of the questions I would have is how we deal with the existing users of OpenSim. Some of these have standalone sims, some have sims on various grids. Of these sims, some use SQLite, some MySQL and some MSSQL.

Hi,

I propose to separate the region specifc parts of EstateSettings
into a RegionSettings object, as a sibling in RegionInfo. This would
read defaults from estate_settings.xml and create records in the
region store, allowing these settings to be modified on a per region
basis.

Existing installs would use their current settings as default, and
installs using no database would continue to work as before.
