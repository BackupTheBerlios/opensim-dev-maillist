Kyle Hamilton wrote:
> Perhaps a better question would be, "why are all of the database
> engines defaulting to MySQL when that requires more work to set up for
> a working installation than SQLite?"
> 
> I'm not dissing MySQL.  I'm simply trying to understand why there's
> defaults to a non-default component.

My understanding (which is somewhat assumed) is that these are 
defaulting to MySQL because running a grid implies a lot more regions 
and users.  By the time SQLite stops being able to handle the load, 
there's quite a bit of data which it has been awkward to port over to MySQL.

I don't think it's unfair to make it a little more difficult to set up a 
grid, so I would actually be somewhat +1 on making the grid config MySQL 
as well.  Either way, we should make things consistent.
