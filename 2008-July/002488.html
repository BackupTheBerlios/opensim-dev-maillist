Hello all,

as an aid to migration, the new estate code contains code to read 
the estate_settings.xml file.

If that file contains corrupt data, settings changes don't persist. 
It seems like it is not a good idea to keep that code, since it 
would likely lead to problems further down the line.

A removal of the xml file was already pretty much a given, I merely 
propose to accelerate this to avoid the issues we have been seeing 
in #opensim.
