Basically, the xml2 format is just a rearrangement of some nodes, to make loading/saving a bit easier.
 
We have been discussing a xml3 format in throes and fits, but never got anywhere.
 
One of the properties of xml3 would be (optionally) not to include id's at all.
 
Then such a load/save file would be truly portable.
In Tribal Server, we actually support loading content off the web into regions, and we have to re-set all the guids in all the objects on each fetch, because they are fetched as new objects into new contexts, and owner et c has to be re-set on all nodes.
That is what really should be the base scenario for loading/fetching, ie, no id data at all.
