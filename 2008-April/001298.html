Hi, Krinke,
It was a very nice talk with you on IRC,
I would like to put my answers here.

Perl UGAI is another implementation of UGAI.
Now it already has critical functions of the C# UGAI:
1. you can now use Perl UGAI + Region to start a VW service
2. It supports: user login, get assets, operate inventory
3. It does no support: friend list, group ,. ...
I am now continue working on it, and aim to implement a full functional
UGAI.

about your concerns
>Are we going to have two sets of servers for grid mode?
>One written in C# and the other written in Perl?
>Will we have each of them with the same features?
>Will we abandon the C# UGAI servers?
you can take "UGAI" as a set of standard protocol (APIs) .
and we can use any programming language which we prefer to implement it.
So, there is no "abandon", perl, C# (maybe php, ruby) are options.
I like perl, I use perl. you like C# , you use C#.

additional comment:
* Perl UGAI is based on Apache, so it is much more reliable(stable) if 100s
of users
login simutaniously. (though, to say "100s simutaniously" maybe a little
early in this stage.)
* Apache provides APIs for many different languages(perl, php, ruby, java
...). Perl UGAI may
extend the scope of OpenSim's users/developpers and break the wall between
VirtualWorld and web ( we could work together ! )

>How will we seperate out issues between regions on a grid and the grid
servers
>if some grids run with Perl UGAI and some run with C# UGAI?
Since UGAI is under standard XMLRPC, you can freely use Perl U + C# GAI,
or anyway you like.
For bugfix or upgrading, I can and would like to provide full support for
it.
actually, I would like to continuously maintain the code.
