> The above is confusingly worded. You make it sound like you need to

> OpenSim that's running master code, and then you say the opposite, below.

It's all academic anyway.  It turns out that through a quirk of fate, older versions of OpenSim can load the changed 
object serialization format after all.

Therefore, I've removed the --version switch from master again and just bumped the OAR version to 0.4 instead, which 
signals backwards compatible change.

However, I wouldn't be surprised if the object serialization format is changed in a backwards incompatible manner in the 
near future.
