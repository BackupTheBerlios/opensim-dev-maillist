mixing up mono versions can give that kind of compile errors.

There appear to be mono 3.0.6 beta rpm packages for opensuse.

Use these rpm packages, compiling mono youselves is tricky. Setting up
multiple vesions of mono on the same distro is even harder. When you have
got multiple mono versions on your distro then there is absolute no
garantee opensim will use the desired mono version even if you have set
things up so opensim should do so, since it is unusual people have got
multiple versions of mono on their system.

I would simply install a new opensuse and make sure mono 3 is the only mono
on that system installed with rpm's.(by using the rpm's you can ensure mono
will installed on your system like it will be on other systems.)
It that case you know for sure error message are not caused by your  mono
install but somehow by opensim being incompatible with 3.0
