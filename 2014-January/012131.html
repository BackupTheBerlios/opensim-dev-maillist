Background:

Kokua maintains one viewer for use on OpenSim and SecondLife grids.


 Linden Lab has continued to refine Server Side Appearance and recently
removed support for client baking. Additionally, AIS V3, an inventory api
has been deployed.


 Kokua has available viewers that have client baking returned and use AIS
V3 features on simulators that have that it included.


 Linked below is a viewer that includes client baking and AISV3 api
included.

http://sourceforge.net/projects/kokua.team-purple.p/files/KokuaSSATest/Kokua_SSA_AIS_SL_and_OS_3_6_13_31056_i686_Setup.exe/download


 This viewer runs fine on OSGrid which is at:


 OpenSim 0.8.0 Dev r/24087 8b3a436: 2013-11-29 02:51:35 +0000 (Unix/Mono)


 It crashes on my local grid which is at:


 OpenSim 0.8.0 Dev r/24130 00e632e (Win/.NET)


 The viewer runs fine on SecondLife beta grid and the AISV3 api works on
the 4 test regions with AIS active.


 The relevant change from SecondLife that exposed this crash is:



https://bitbucket.org/NickyP/kokua-ssa-ais-sl-opensim/commits/72bc60181c12630c56512b39768bb38822e19c40


 It may be some time before this becomes an issue but, I wanted to report
while it is narrowed to

43 master commits between works and doesn't.
