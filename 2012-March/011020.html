Hi everyone,

I was wondering if it was possible to use dynamic hypergrid addresses, and
if not yet working, is it a lot of development ?

Let's consider that we have some standalone in the cloud or on any other
infrastructure and that it may change its IP address very often for some
reason and that it's not suitable to change DNS records all the time.

For example, I wish to reach a region on a foreign standalone which HG
address is : *12.34.56.78:9000:TargetRegion* (or 12.34.56.78:8002:TargetRegion
if we were talking of a grid)
Would it be possible that I set an 300 HTTP redirection (or using a PHP
location header, or whatever...) that redirects a fixed HG address like* **
www.mywebsite.com/hg-addresses/targetregion*<http://www.mywebsite.com/hg-addresses/targetregion>
to
the dynamic Hypergrid address above ?

This kind of implementation would be great because, HG addresses would
become web addresses that anyone can easily remember and use to visit HG
regions.

Thanks for your consideration,
Best regards,
PMint.
