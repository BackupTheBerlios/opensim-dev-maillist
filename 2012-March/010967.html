> Omg many terrabyte file system, that is a lot. Hoever opensim databases
quicky grow to multiple gigabytes ...

And as the opensim worlds are user generated worlds, lots of people do not
really care about things like small textures or small meshes.

In the virtual words assets there are a few cases in wich a lot of data has
to be pushed. When users are opening their inventory and when they arrive
at a region. (in case of events sometimes 20+ people are arriving at the
same time)

When the assets are small in size, maybe less than a 100k then it would not
be issue if they are stored in a database. But when a database has to push
lots of mb's (a heavy textured and meshed sim can be a multiple of 100mb's)
than it would probably be better when the assets are fetched from an file
somewhere in a file system (maybe even on another server to preserver
bandwidth)

I think it would be good when large assets are stored in a filesystem, and
there is just a refence to that location.
