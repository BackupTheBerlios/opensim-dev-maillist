I have to disagree.  Using file storage seems a last resort, not a first 
resort, if you have a database with indexed tables available.    Good 
databases usually solve the problems of file-system storage,  not the 
other way around.

Several times in my last ten years at work we considered supplementing 
Oracle with file-service, for things like images, and we always ended up 
rejecting file system solutions.

M.E. -- can you explain in more detail how you figure that file-system 
storage and retrieval and deletion would be faster and more robust and 
scalable than MySQL calls for the same amount of data?
