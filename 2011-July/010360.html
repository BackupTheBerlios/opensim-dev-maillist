In the [Startup] configuration parameters there is an AllowScriptCrossing which is false by default.

To the best of my knowledge, this just controls whether a simulator running on another machine reloads script state 
data.  Since the script would still be restarted anyway (just without state), is there any good reason for having this 
default to false?

Also note that this is very different from the TrustBinaries setting, which would be very dangerous to set to true. 
Though having said that, on reading the code I can't see that saved assemblies are actually being read anywhere.  I may 
be missing something.
