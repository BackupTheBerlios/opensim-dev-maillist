The ROBUST server is not a general purpose web server and a real web
server is much better suited to this task. You have to specify a
-loginpage anyway, so there is no requirement that this be the same
address as the login service.

That said, Robust services are not rocket science. If someone wants
to create a Robust service that acts as a normal web sever serving
static text files, then that would be a candidate for inclusion.
Personally, I think this is not required functionality and while
there are bugs to fix, I at least will not have time to reinvent the
wheel and implement serving static html pages in Robust.

Melanie
