There have been no changes to the RemoteAdmin (standalone) XMLRPC createuser functionality.

However, I have heard an unconfirmed report that it is not working.  Can anyone confirm/deny these?  Chris, if this 
functionality is important to you I would also strongly recommend that you test your createuser code against 0.7.4 and 
current master and raise a bug report if appropriate.

There may be some confusion because 'createuser' refers to both the standalone RemoteAdmin facility and a ROBUST service 
call available on the private ROBUST service port (off by default).  The ROBUST version was not available in 0.7.2 so 
you must be using RemoteAdmin.

These calls do the same thing but have largely different implementations.  The RemoteAdmin version has some additional 
code that is meant to provide flexibility in setting up default avatars but I'm not sure this is working either.

In the long term, I don't think it makes sense to have both of these.  The code would be more consistent and less 
complex if the service call was also available in the standalone configuration.

However, I believe this would require the service call to have more security.  So I don't think it won't happen 
imminently - I would fully expect this to be available in OpenSimulator 0.7.5 and probably for a considerable time 
beyond (with the caveat above about whether it's currently working).
