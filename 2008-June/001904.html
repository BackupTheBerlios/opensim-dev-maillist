Hi,

Just committed plugin support for ClientStack, with LindenUDP as the 
default plugin (r5161). This makes it easy for developers to experiment 
with alternative communication protocols. I'm doing some experiments 
with a new libSL implementation internally at the moment.

I added a ClientStackManager that loads the plugin, and changed 
LindenUDP to be compiled as a DLL. Since the LindenUDP API is unchanged, 
I don't think it will cause any problems.

/Johan
