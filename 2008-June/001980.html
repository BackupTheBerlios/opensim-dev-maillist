Again, I don't see why we need to have assetId === sha-1 hash - if the only thing we want to accomplish with it is de-duping, we just make sure to send the hash along, and any context can choose to use it to de-dupe or not.
 
And to trust it or not.
 
Obviously, you wouldn't trust an assetId generated by an untrusted source to represent the asset binary anyway, so quite a lot of re-hashing and checking will be done along the way anyway.
