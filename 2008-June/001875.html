What's on the table right now would allow scripts to specify an API 
  list (by virtue of .NET ref'ing).

If an API that is used by the script is not there, the script won't run.
Tedd and I spoke about this last night, and there really is no 
sensible way at the moment to "fake it"

So, basically, scripts will pull in what they need, and fail if they 
can't. Either because it's not present, or because it's forbidden by 
configuration.
