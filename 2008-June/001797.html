Right now UUIDs are used as a key to everything. As far as I can see
these UUIDs are unique only on one single grid - on another grid the
same UUID could point to something completely different. Thus,
connecting two grids won't work. As far as I can see the realXtend
people do the travelling from one grid to another 'simply' by loging out
the current user and loging in a new user on the grid you travel to,
thus needing to create accounts on all grids you want to travel to.

Just a small correction; in realXtend the avatar portability is handled by the avatar authentication and storage services, which are independent of the grids, but only when traveling between compatible grids. For Second Life(tm) you still need a separate account, but you can move freely between realXtend grids as long as the grid operators haven't blacklisted your authentication service. All the avatar data is stored in a separate storage and can be fetched from there when necessary.
