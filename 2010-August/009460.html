I find the megaregions awsome. There is no delay when crossing sim borders
and I have found 1 megaregion is less laggy than 16 normal regions. It shows
all the regions at once. So no grey textures when I fly from one corner to
the other.

The setup of the regions for a megarion is tricky though. There does not
seem to be a warning when the regions are not set up correctly.

When you have got a 2x2 region lets say:

cd
ab

then you can add anther region, but the a region has to be the lower left
region (the root region)

so you should be able to add reigons like:

cdy
abx

xy
cd
ab

but not:

ycd
xab

cause this would change the root region, the region where the prims reside.

I have not tried adding a singe region. In theory that should also function.

I am not sure if it is important, but I set the regions in correct order in
the regions.ini file first a then b then c then d.
When adding xy like

cdy
abx

Then I would order the file like a then b then x then c then d then y.

The root region makes sence, since negative prim coordinates are not handy.
A prim in region y could for instance get coordinates (600,600,30)
