Well, to be completely correct the prim/object references the users
key/uuid.  Right now that's resolved to the account name when its
displayed.  And that name as proposed would stay the same.  But I may
choose to use standard Display Name all the time and build my identity
around that. Thats far more likely in fact because in the future account
names will be a single entry  (aka mdickson.resident) rather than first
name/lastname format.

My point is, how names are handled is really up to the agent service.
This can and probably will vary across grids.  So caching it in a
globally unique id has no real value or meaning. If OTOH, OpenSim is a
product and as a product you want to dictate the semantics of names then
you can lock them down to mean one thing. But I'm pretty sure thats not
what's being proposed here.

Mike
