I agree!

For the overhead of handling the well known problems of cacheable_data (e.g.
locking during updates) this solves every objection that I can think of.

Most importantly this ensures that there is always a resolution for the
identifier.

A simple history file (journal?) can track changes made to archived
identifiers (i.e. a list of changes made to the cache - analogous to a file
system journal). As I think of it there are several ways to handle this
problem, all of which fit nicely within the general concept that you have
made.
