> Having thought about it a bit more, I am less keen on the idea of bundling a
> web interface now - as well as being a support issue it is an extra piece of
> stuff that people using another web interface would need to remove or
> ignore.

My thoughts are somewhat simple
1 have a set of standard "connections"/APIs in the base to link into
2 make sure that any bits bolted on have a nonconflicting namespace
3 ideal would be for a module to add/remove "Clean" (everything in the
modules own folder and any needed mods to normal config files to be in
a single section)
4 any "stacked" modules need to have a copy of the entire stack down
to base on the same website (so an addon to wifi should be either
posted on Divas website or the base wifi modules should be posted on
the addons site)
