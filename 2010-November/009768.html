Aloha,

This is a serious failure.   There is NO documentation anywhere to direct
anyone to the proper fields/mechanics on how to assign an estate manager and
there are many basic features that depend on having estate privileges.

Personally,  I fail to see why the inclusion of estate manger fields in the
old INI, files  ...and later,  the prompting for an estate manager name when
a "new estate" was created during region creation, was deprecated...

IF the mechanics of estate management were so significantly BORKED as to
require a re-design to work with the .7 refactor, then I would have expected
at least some period of discussion about  what the implications would be.  I
didn't see that, at least in the mailing lists I have time to follow.  I
also seriously doubt this is the case,  since I can point my version of the
"slim-updates" ( pre .7.0.01) release at my current database and use the
prompts in the  region create process to populate the estate manager field
properly.

Posting a response like  "NO"...   well it certainly doesn't advance the
projects reputation for attempting to support valid feature requests...
Maybe  a hint as to what the related issues are,  and the possible
directions an interested party,  (who has just self-identified) , might
pursue to enable such a feature  would be more productive?
