Just chipping in my 2c: How about the modules are supposed to register the 'command' whatever that is, but that the 'command' can include spaces? So, "create user" should be a valid command, sending the remainder to that region (via the registered delegate). which means another module can register "create region" but not "create user", as that would have already been registered. Of course, it cannot register a non-character terminated subset either, like simply "create". the command parser would simply run thru the list, doing a 'starts with characters' match, probably with a non-character terminator in the regexp. (If you're really anal on performance, the whole list of matches can be concatenated into a regexp and compiled once) Of course, you would have to register "help create user" as well, but it seems to me that there could be an abstract base class providing some protected generative helper functions.
