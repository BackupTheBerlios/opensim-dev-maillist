Hi folks.  Over the last couple of months, we've had many reports of simulators being unstable if users login before 
scripts have finished loading.

In commit 9d458fdd today, the [RegionReady] module is set to enabled = true and logins_disabled = true, which will 
prevent this.  This setting has been enabled on the osgrid distribution for some time without reported problems.

If you want the old behaviour, please copy the [RegionReady] section from OpenSimDefaults.ini into OpenSim.ini and set 
enabled = false.
