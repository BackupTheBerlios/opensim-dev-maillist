First : I run the latest build svn.6008.

- Startup seems to be faster.

- After the grid startup, CPU is no more 99%, but about 50/70%
- First login : CPU Still ok, 50/80%
- logout : CPU is 99%
- Second login : CPU back to previous values ( 50/70% )

Another point :
- Script engine not working, cannot compile even the default lsl script.
- During the second login, I got many errors on the Region console,  
but seems to be ok, because the system is overloaded.

When I use the htop utility, discovered that only one "OpenSim.exe"  
thread is consuming 99%CPU ...

One information for all, I know this server, based on a Via C7 CPU, is  
not powerful enough to run 25 Regions full of prims. And I have read  
so many bad things on this CPU over the internet :S

BUT: it's always interresting to know the limits, it's the first of a  
serie of stress tests I will run.
For now the goal is to answer the question :
--> How many regions can I host on my server, regarding to RAM/CPU  
Cores, ...

I plan in the future to try to reach the 15000 prims limit, just for  
fun :) just to stress both hardware & software.
It's always interresting to know, specially when the product  
( opensim ) is nearly "mature" for production.
A benchmark plan is always needed to improve stability & performances.
And I know it takes many hours to run and analyze ( it's my job for  
now :P ).

Am I right ?
