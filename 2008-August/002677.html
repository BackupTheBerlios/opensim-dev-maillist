> Hello All,

> We all know Opensim software start to be more and more popular and that's a
> good news but unfortunatly our weakest point is the lack of "serious
> testing".
> 
> We provide a stable release and an up to date SVN with all the limitation of
> a development branch.
> 
> My proposal is something in between:
> 
> 
> Proposal
> -------------
>   We could create a weekly load, based on each Friday commit for exemple.
>   No Commit should be perform during this day to avoid any mismatch.
>   During this day, we perform a tesplan (included in this mail) that will
> check the non-regression part of the build and the feature we want to
> provide.
>   After the tests, we create an archive open to the public with a small
> detailed release note explaining the new features and some special
> parameters if needed.
> 
> This Proposal will help to provide some accurate release "ready for
> production" and should reduce the mantis volume due to outdated release or
