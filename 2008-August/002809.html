Frisby, Adam wrote:
> I’d like to start collecting automated stats (you know those 30 minute 
> things that show up on the region console) going to opensimulator.org as 
> a “opt-in” thing.

At the moment a lot of the stats (server oriented) which actually show 
up there are pretty random (added to enable certain things to be 
debugged at various points).  I'm not sure how useful they would be, 
though of course that doesn't stop more useful stuff being added.

Of course, this is separate from being able to send back client stats 
(as Teravus talked about in another post).

> 
> There’s a number of issues coming up now where having some detailed 
> long-term profiling information could be handy. Does anyone have any 
> problems or suggestions here?

Any idea what kind of things might be useful?  An interesting start 
would be hardware information, number of avatars logged on and sim fps. 
so that we could get a handle on the old "What hardware do I need to run 
OpenSim?" chestnut.
