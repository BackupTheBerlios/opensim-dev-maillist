> I'm currently about to begin a project on OpenSim for a project at University. I'm looking at an export/import tool.

> I noticed the OpenSimExport project for the Google summer of code and wondered if anybody is currently working on this? If not it would be more useful if I work on it rather than work on a completely independant project.

> If that is the case, what would people want from this tool? I am limited by time, so I probably won't be able to complete it fully, but I would like to have the core functionality finished. I notice Sean was the one who proposed it so I'm guessing you have an interest in this?

Look at the save-oar/load-oar functionality and tell me if this is what 
you're talking about.  This currently saves out and reloads a tar.gz oar 
region archive (prim xml + the assets needed to restore this prim + 
region terrain).

The export tool is something of a shell at this point, though it might 
be nice to get it to be able to load/save archives without needing the 
opensim region to be running.  This will involve a good bit of 
refactoring to isolate the components of OpenSim that need to be reused.

One thing you might think about adding is inventory archive 
loading/saving, which is basically the same thing as for regions, but 
with inventory :-)  I made a start on this but haven't had time to make 
much progress recently.
