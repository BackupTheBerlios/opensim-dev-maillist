Hi all,

1.9.1 is slightly different than 2.0 pre1 and pre2, and these in turn 
are slightly different to 2.1 which is the working version for the head 
of mono SVN.  We've run tests on all of them, and I must say it is 
getting better and better as the mono project walks along their 
development path. In particular, the logging profiler became part of the 
standard package, and profiling as such is much more stable than before. 
Well, it still does not work most of the time with OpenSim.exe, although 
the good news is that it hangs or crashes much later than before ;o) 
We have had some successes getting other UGAI servers profiled. 2.1 
seems to work a bit faster, and uses less memory, although GC is still a 
bit of an issue (parallel marking and sweeping and incremental GC are 
theoretically possible but still  problematic). We are tweaking the 
compiling options and GC setup as to squeeze better performance out of 
the system. Ahead of time compilation seems to help too, although for 
exact quantitative numbers we need to put more work into it.  My gut 
feeling is also that both 2.0pre and 2.1 are more stable (less OpenSim 
seg faults observed).   Valgrind still hangs on most of the UGAI servers 
(OpenSim.exe in particular) but we have been able to get some memory 
leaks profiled - there are some - more work needs to be done to isolate 
exactly what's causing it - and how this can be avoided by tweaking C# 
code.

cheers
Mariusz
