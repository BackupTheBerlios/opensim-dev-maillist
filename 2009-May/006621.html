On 2009-05-25 17:04:24 -0400, robert at rd-it.net wrote:
> OpenSim 0.6.5-post-fixes according to subversion is revision 9668
> but when I run it, it displays this:
> [STARTUP]: Version: OpenSimulator Server  0.6.5.9673
> 
> So what revision number should I write (9668 or 9673)
> when I find a bug in this version?


If you're building from a branch, you should indicate both the branch
and svn revision you're using, in this case branches/0.6.5-post-fixes,
revision 9673.  If you're building from a branch, then an svn revision
number alone isn't enough info to identify what code you're running.

-Coyle
