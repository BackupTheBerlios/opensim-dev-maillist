So, since this was thought to be a good idea, I just copied the branch
 
/branches/0.6.0-stable from the tag /tags/0.6.0-release.
 
This branch is _only_ for applying post-release stability fixes - ie, no new user features, however minor.
 
Of course, this is a grey area, but I think you understand what the use of this branch is.
If somebody fixes something for trunk, whoever feeling inclined, should re-apply it to this branch as well, marking it clearly that they have done so.
 
We should be able to refer to this branch as "the most trustworthy revision for this version" at all times.
 
The difference form the 'tag' is that while the tag is 'stable' as in 'we know what's wrong, and you can trust us not to try to fix it', the branch is 'stable' as in 'if you find anything wrong with the tag, we might have fixed in the branch'.
