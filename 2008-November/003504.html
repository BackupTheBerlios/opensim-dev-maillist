Well, that may make sense for some clients, but not others.

Right now there's really two classes of virtual worlds in action today:
        - Those with Scene size limits (SL, etc)
        - Those with infinite 'rooms' connected via portals (Croquet, etc)

I see it as being fairly useful to be able to represent those 'room' worlds using OS as the base.

If you are using voxels/etc for proximity, I imagine you would do so in terms of a module rather than a scene-level change.
