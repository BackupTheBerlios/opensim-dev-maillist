Dahlia Trimble wrote:
> Wouldn't this possibly flood mantis? Maybe it should go to some other
> database first and then go to mantis after it's been sorted and tabulated so
> it wouldn't generate duplicates?

That's my only concern, that we'd end up with a flood situation.  I
think this should go into a separate mantis tracker, as the ways you are
going to want to filter these are going to be a bit different from the
hand made reports.

I suggest making the final approach here something like the crash
reporters that many pieces of software have
http://www.nationmaster.com/encyclopedia/Crash-reporter which includes
popping up a window and getting some user feedback on top of the crash.
