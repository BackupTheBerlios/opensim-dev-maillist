Thanks.

I would expect to use enumerate mostly for garbage collection (not that
inventories ever get out of sync with the asset db) & external backup
and object migration.

The import/export is more interesting if the asset server can be
accessed directly by the user. It would be nice to be able to hook up
3rd party content creation tools that can import/export something like
collada or obj format. In that case, a single UUID will reference the
object independent of its format. 

Neither function is necessary for 0.5 (so long as the 0.5 design doesn't
preclude adding it later).
