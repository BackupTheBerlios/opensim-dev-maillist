Off the top of my head (and glancing at commit log) we've done the following:
 
* changed hardcoded /bin/Debug output directory to accomodate for our /bin structure
* worked quite a LOT on various file path generation bugs (the way Prebuild did it then was just SO broken, they might have fixed it now)
* being able to pull in external nant build targets
* being able to reference .exe files
* fixed node output order that would differ between mono and .net due to differieng enumeration of 'hashtable' which led to difficult-to-track cross-platform breakage.
* make sure projects always retain uuid between regenerations of vs targets, so external properties (like 'startup project') stayed the same* introduced vs2008 project
If somebody with a foot in the prebuild community would like to work with re-synching us with upstream, we'd all be very very happy, I guess. A big wish from the vs coders would be to be able to specify 'project folders'; I've been meaning to code that for like 4 months now.
