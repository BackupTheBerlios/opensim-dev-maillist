As one who runs in standalone only, for content previewing mostly, I echo
Ter's request to keep it simple:

Install
Start
Use

All working defaults with any complexity abstracted behind whatever you want
as long as it is still as stable and easy to manage.

I agree with the simplification of code into everything-uses-asset-server,
or a grid-of-local-one approach but am frankly leery of inadvertently
exposing greater complexity not just at startup, but possibly during
management of the running processes. The reduced time wasted with code from
simplification will probably net out very shortly with the work to
encapsulate complexity, so still for grid-of-local-one approach. It might
even simplify configuration files somewhat.
