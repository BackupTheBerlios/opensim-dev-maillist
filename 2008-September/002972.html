Ok, I think we are now generally agreed on the following approach:

* Tests for an Assembly should be in That/Assembley/Tests - so that they
  are close to the code they test (and now that we know prebuild has an
exclude stanza)
* This will be compiled into That.Assembly.Tests.dll - so that we aren't
 carrying around tests inside production assemblies.

There seems to be broad general consensus here, so unless there it an
objection, I think we can consider this closed and just put this into
action.  As I was the one that first brought this up, I'll take
responsibility to adjust the tree accordingly.
