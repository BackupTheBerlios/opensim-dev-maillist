For the openID - (I think there was a thread around this topic) why not
implement it via an OpenID-aware CGI script that would output a onetime
password to be used by the viewer ? This would be very very simple and
reasonably secure - and, again - loose coupling ?
So then making it an "LDAP" authentication for the enterprise use would be
just trivial - change the CGI to be authenticated against LDAP, and use
still the same one-time-password system there ?
