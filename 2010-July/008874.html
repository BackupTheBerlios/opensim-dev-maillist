There is a very useful diagnostics line in the Opensim.exe console 
log (Opensim.log) hat gives the UUID of the staring script, the UUID 
of the prim its in and the X,Y,Z position f the object 
involved.  Very useful for tracking down issues.

But I note there is no Region stated.. so if you have several regions 
starting up in a single Opensi.exe, then its not clear where the 
scripted object is.

I think Justin may have added this diagnostics line a year or so ago 
when we were tracking down some issues at that time.  I wonder if the 
region name can be added in between the @ and < when a dev is next 
looking at that area of the code?
