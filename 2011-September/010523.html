I think that having a native ROBUST groups service would be a good idea, since it is a very common components.  However, 
I don't have the time to do this myself.  Patches are very welcome.

I also think the approach of having all services run in a single Apache instance is also very good.  Apache/PHP would 
almost certainly be vastly more efficient than the current native C# http server code that ROBUST has to use.  However, 
this is another significant undertaking but it could be performed as a separate project.  People have done this kind of 
thing in the past but I'm not sure if any of those projects are up to date.

As Toni said, there's nothing stopping people creating their own distro of OpenSim with various components added.  The 
whole virtual environment space is immensely complicated with many different configurations and approaches, so I don't 
think that one project could cover them all at this time.

As for alpha state - to my mind there are still a lot of obviously janky things about OpenSim.  Toni has mentioned some 
of the internal ugliness.  Also, OpenSim still crashes with ODE issues under some circumstances, the commands are 
inconsistent and incomplete, etc.  I feel that these issues need to be addressed before one would want to raise 
expectations.
