If I understand you correctly, that's not what we have working Justin.

We run Freeswitch on a server and connect it with a ROBUST remote 
connector for Openvue for all our OpenSim.exe instances (and hence 
regions) on Openvue.  We get one separate FreeSwitch channel per region.

We can also use the same FreeSwicth server to serve regions on ANY 
grid set up to use Freeswitch via the local connector in OpenSim.exe, 
or the remote connector tying across to the Openvue ROBUST service. 
We do that for our own hosted regions on OSGrid.

So if you meant grid wide voice so people can talk across regions..we 
have not done that.  We should fix the wiki if my notes were misleading.
