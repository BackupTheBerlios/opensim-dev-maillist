I believe one key contextual component we have, that most 'web' scenarios don't have, is that we can base our authentication on 'pushing' authentication.

 

I believe that it should be very possible to create a scheme where you always start your session with logging onto your home registration grid, then establish a viewer session with each region in turn.

 

Now of course, something like openId could probably be used intra-session. My only beef is that it should never be considered main provided 'entry' authentication scheme, for what I consider obvious reasons.

 

So, if we can let 3D Web resources be so portable (continue down the separation and distribution path started upon by hypergrid) that you only ever need to register one account for your OpenSim experience, and if there can be third-party providers that provide openId entry authentication that lets me use openId for that (me knowing the risks involved, but trusting this particular party) I think that would solve the whole thing.
