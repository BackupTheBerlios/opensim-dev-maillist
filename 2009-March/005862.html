Visual Studio 2005 is using .NET 2.0/C# 2, only with some extending can you use some .NET 3/C# 3 features (WCF is the best known).

It seems that Visual Studio 2008 is doing some tricks for you when you have a .NET 2.0 project, but use .NET 3/3.5 syntax. Like this:

public string Test { get; set; }

Just works in a .NET 2.0 project, same with lambda, while it is .NET 3/ C# 3 specs
