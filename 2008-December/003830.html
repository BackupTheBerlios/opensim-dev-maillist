Hurliman, John wrote:
> The GPL license requires that any code derived from GPL code must also be released under a license compatible with the GPL, so you can't take GPL code (or possibly even ideas, a well debated nuance) and release it under a less restrictive license such as BSD, MIT/X11, Apache, Mozilla Public License, etc.
>
> The mantra of the BSD license is that writing code and seeing it used in successful projects is reward in and of itself. There is no hostility toward commercial vendors or alternative forks, and everyone gets along peacefully.
