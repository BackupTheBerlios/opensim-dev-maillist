Just as a point of information, the contents of each of the config files
are read as a stream in a particular order. The order assures that some
settings in some files can 'over-ride' those set in files read earlier in
the stream (e.g., OpenSimDefaults.ini is read first; consequently, things
set there can be over-ridden by settings in files processed subsequently
from the stream). There are others that can speak more directly to the
specific order in which they are read; for myself, I tend to stick to the
stock configuration as much as possible (it simplifies updates greatly),
and so don't take advantage of the inherent stream architecture.

While there are certainly 'best practices', especially with respect to
recommended/supported configurations, the operator is free (encouraged?) to
take advantage of the configuration file architecture to bring
configurations into line with site operations policy, etc.

So by all means experiment but be aware that everything you do will have
consequences, so keep an eye out for those that are not intentional. ;)

Cheers!
