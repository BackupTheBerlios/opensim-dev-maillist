Robert Martin wrote:
> In pondering the subject i was thinking why isn't there an opensim.ini
> file included in the package (with that name)?
> or Why is the file given the name opensim.ini.example??

Top of my head, 2 reasons

1)  If OpenSim.ini exists then it's very easy for developers to accidentally checkin changes they make to this file

2)  Having to copy opensim.ini.example provides a strong signal that people then need to look in this file and think 
about what changes they need to make for their own configuration.  Changes are always necessary unless you want to run 
an sqlite OpenSim standalone.
