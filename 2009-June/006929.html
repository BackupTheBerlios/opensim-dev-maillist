I'm ambivalent about doing separate repositories for the code that's
already in the main OpenSim repository, but the scenario I'm talking
about can all be done with the current, single repository for that
code.  It's simply a matter of file and build structure.

If someone only wanted the core simulator code, they would checkout
http://opensimulator.org/svn/opensim/trunk/OpenSim

They could then check out individual modules from
http://opensimulator.org/svn/opensim/trunk/OpenSimModules/

Or if they wanted the whole nine yards, they would checkout
http://opensimulator.org/svn/opensim/trunk
