I think you probably need to be a core developer to do this :)

But generally speaking I would support this kind of release process since it shares the release pain more evenly over 
all the developers rather than pushing it onto one or a few people, as tends to happen in the branch approach.  The 
quicker that showstopper bugs are identified after a release candidate has been set, the quicker the release proper can 
be put out so that everybody can get coding again.

However, it will require a strong personality who is prepared to revert code in the rc period, and I think that this 
process would require the consent of all the core developers.
