Teravus did some good investigation work over the past few weeks and
very helpfully suggested that we change the way nant calls nunit to be
exec based instead of using the built in nunit task.

It turns out the built in nunit task uses nunit 2.2 (not 2.4), and has
been known by many projects to be flakey.  Since we switched over last
night we haven't seen any more errant build breaks (though only about 10
builds happened since then, so too early to say it's solved for good),
but it does look promising.

Hopefully this means panda is a bit more accurate in it's assessment of
good vs. bad commits.
