Right, all changes are only aplyed to the module tables, not core ;)

Reason for deleting old data is, that if you try searching in osgrid you 
will find most entries havnt been existing for ages. The search module 
does however regularly update the search data were it already removes 
all data of the current region prior to readding the new data. I also 
used 7 days as a variable sothat its easier to ajust to gridspesific 
demands/wishes. The search should however be as up to date as possible.

If a region is running longer than 7 days, then it should have 
automaticyly updated the search atleast 1 times already... thus the data 
will not be removed by the 7 days limit. The data update is triggered by 
parser.php which sends a message to the simulator... I chose 7 days as 
this would atleast keep the data of those people that have their Regions 
running every weekend.
