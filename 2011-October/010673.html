There are many opinions here, including ones which sound like stating the priority for the
implementation. However, actually, no one can decide the priority. Thus, the answer to your 
question would be,

"When someone who can implement it notices that implementation of the display names is
important."

Personally, I, as a SL or virtual world roleplayer, will be very appreciated if we have the
display name in OpenSim-based grids. We don't need to use scripted titler, name-changer 
or like if we have it. Thus, it will reduce the total amount of scripts in a sim and green-characters!
I bet some people thinks it important as well. I doubt they are in this dev mailing list...

I wish I could implement it, but I'm now occupied with other matters than OpenSim project and
 can only act as a spammer buster in wiki.

At a glance of current codebase, it will first need to add a column to user table in each type of
database. Then, we can refer and update the value through common accessor. It might be a
bit more difficult than simple fix for script functions or physics calculations.

If you are deadly need to have this feature now, you might want to look for some "folks" project,
which may have already implemented it.
