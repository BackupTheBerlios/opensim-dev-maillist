> About the only way to do a (near) realtime whiteboard that works with the
> standard secondlife client, is to do it like the prototype I did.
>
> Which basically embedded a Video streaming server into a opensim module. So
> you set the parcel media url to point to that video streamer. Then the
> module could handle the click/movement comands from the client (hooking into
> the opensim IClientApi events) and then in "near realtime" draw on the
> surface that the video streamer was sending to the clients. It also included
> a VNC bridge, so the module would act as a VNC client, and converted the VNC
> desktop into a video stream.
>
> As Stefan said, I wrote a prototype doing this, but it was just that... a
> prototype and lots of things would need doing on it before it was really
> usable. I've often meant to post the code to the forge but the biggest
> problem is that the code is mixed in with a lot of other custom code, so
> extracting it is going to take quite a bit of work. Which is time that at
> the moment I just don't have.
