Adding support to OpenSim for some basic mesh types may be rather trivial.
I've done some initial experiments with compressed .obj files and the
existing asset system and have had success. The problem isn't really with
OpenSim, rather it's the LL viewer. Realxtend can use more mesh types than
the LL viewer and a module exists for OpenSim to support these capabilities.
IdealistViewer can display many mesh types simultaneously with prims and
sculpties, but development is slow. More contributors would help to bring
these capabilities to OpenSim and Idealist sooner.
