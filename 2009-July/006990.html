Getting this to work effectively will require several changes
(including database changes) and a little bit of luck dealing with the
client.    Particularly, older clients have issues with the media sent
over the eventqueue like is required for the web content on a surface
and crash almost immediately after login.   Feel free to write a patch
for it.  If you do, it is suggested that you have a script that
changes the parcel media to verify that scripting it works correctly.
It is also suggested that you have an older version of a client to
test against.   Some people still use clients that do not force the
use of the EventQueue.  You can reproduce the crash issue with version
1.18.3x until 1.19.x with the parcel media sent over the event queue
with the extended data that includes media type.  I believe that the
LibOpenMetaverse developers would also like to see us finish this so
they can phase out some code that handles the parcel media event queue
message without the extended data block.
