Yep, that's it.Those events are there for being used.When writing region modules just keep in mind that your region module code runs natively with the core code; if you mess up, the core behavior may get messed up too. So, for example, if your module adds significant delays in processing an event, it may interfere with the normal OpenSim operation. So be smart with your region modules :-)
